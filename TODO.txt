Modrinth — Manual Test Checklist
================================

[ Browse page (/modpacks) ]
- Navbar shows "Modpacks" between Templates and Status Page; dashboard has a
  "Browse Modpacks" button next to Import. Both land on /modpacks.
- Page loads with popular modpacks (relevance, no query). Cards show icon,
  title, author, 2-line description, compact download count, loader badges.
- Card hover elevates like server cards; icons load from cdn.modrinth.com
  with NO CSP violations in the browser console (check DevTools).
- Search box debounces (~400ms) and updates results; empty query works.
- Loader filter (All/Fabric/Forge/NeoForge), MC version filter (populated
  from vanilla version list), and Sort all re-run the search from page 1.
- No results -> centered "No modpacks found" empty state with search_off icon.
- Pagination: Previous disabled on page 1, Next disabled on last page,
  "Page X of Y" correct. Filters + page survive in the URL — use browser Back
  from the create page and confirm you return to the same results.
  (RETEST — fixed: page total no longer advertises pages beyond Modrinth's
  10,000-result offset cap, so the last shown page is always reachable and
  the count reads "Page 556 of 556", not "of 557".)
- Quilt-only packs do not appear (default and per-loader searches).
- Clicking a card opens the version modal: spinner -> version table with
  version number, MC badges (first 3 + "+n"), loader badge, published date.
  Packs with no compatible versions show "No server-compatible versions found."
- "Select" navigates to /servers/create?modpack=<id>&version=<id>.
- Stop the app / block network -> search shows danger toast + failed state.

[ Create page — modpack mode ]
- Arriving via Select: template/type/version pickers hidden; modpack summary
  card shows icon, title, "Version x.y.z", loader + MC badges.
- Server name pre-filled from pack title (filtered to allowed charset);
  memory bumped to 4096 with Advanced auto-expanded and the "Modpacks usually
  need 4-8 GB." hint (only if memory was below 4096).
- (RETEST — fixed) Fields alone on their row center: the port field in
  modpack/mrpack mode, and the Assign Group field in Advanced (all modes).
- The X (btn-close) on the summary card returns to the plain create page.
- Create Server button stays disabled until name/port/memory + EULA are
  valid, exactly like the normal flow.
- Submit: button swaps to spinner "Creating...", overlay 'Setting up "<pack>"...'
  appears, then redirect to /servers/<id>.
- Invalid modpack URL params (e.g. ?modpack=abc&version=def) -> "Failed to
  load modpack details." toast + bounce back to /servers/create.

[ Install pipeline / progress ]
- After redirect, console page shows the provisioning banner and it ticks:
  "Downloading the modpack..." -> "Reading the modpack manifest..." ->
  "Installing the mod loader server..." -> "Downloading mods (12/87)..." ->
  "Applying modpack configuration files..." -> "Finishing up...".
- Refresh mid-install: banner shows "Running first-time setup..." then live
  progress resumes on the next progress event.
- On completion: success toast, state flips to Stopped (no reload needed),
  server icon becomes the pack icon, Mods tab shows the downloaded mods,
  server.properties has YOUR port/gamemode/difficulty (not the pack's),
  eula.txt present. Event log has "Installed modpack ...".
- Install one pack per loader and START each server:
  * Fabric pack (e.g. Adrenaserver) — boots; fabric loader version matches
    the pack's pinned fabric-loader.
  * Forge pack — installer runs, libraries/ + args file layout, boots.
  * NeoForge pack — same, and the Mods tab now exists (serverNav fix).
- Failure paths:
  * Restart Craftbox mid-install -> server shows crashed with
    "Provisioning interrupted by restart".
  * Kill network mid-install -> crashed with "Modpack install failed: ..."
    shown verbatim in the crash banner (reload the page too — the
    server-rendered banner must also show it).
- Dashboard live-updates the new server card during/after install.

[ .mrpack upload ]
- /modpacks header "Install from .mrpack" -> create page with file input
  (required; Create disabled until a file is picked + fields valid).
- Small .mrpack (<5 MB) uploads via single multipart request; large one
  (>5 MB) goes chunked (watch /upload/init + /chunk requests) — test through
  the Cloudflare tunnel if available. Overlay shows upload % + filename.
- After 201: "Modpack uploaded — installing..." toast persists across the
  redirect; install proceeds identically to the Modrinth path.
- Rejections (clean 400 + toast, no server record created):
  * a .zip renamed to .mrpack that isn't a Modrinth pack
    ("modrinth.index.json is missing")
  * a non-zip file renamed .mrpack ("Not a valid .mrpack file.")
  * a Quilt pack ("Quilt modpacks are not supported by Craftbox.")
- Verify no orphaned server appears on the dashboard after a rejected upload.

[ Mods/plugins for existing servers ]
- On a STOPPED Fabric/Forge/NeoForge server: Mods tab -> "Browse Modrinth"
  (outline-success, travel_explore icon) beside Upload. On a running server
  the button is disabled with "Stop the server to install" tooltip.
- Modal: header "Browse Modrinth — Mods · Compatible with Fabric 1.21.x",
  search + sort work, results filtered to the server's loader + MC version.
- (RETEST — fixed) Installing any mod no longer 400s with "Invalid file
  name" (isPathInside args were swapped; verified with JEI on Fabric 1.21.1).
- Install a mod WITH required dependencies (e.g. Sodium or a mod needing
  Fabric API): toast reports "+N dependencies", both jars land in mods/.
- Install button flips to disabled "Installed ✓"; installing the same mod
  again from a fresh modal -> 409 warning toast + button marked Installed.
- Close the modal after installing -> "N mods installed." toast + page
  reloads with the new files in the table.
- "Load more" appends the next page of results.
- On a Paper/Purpur server the same flow searches PLUGINS (bukkit/spigot/
  paper family) and installs into plugins/.
- Start the server afterwards: reconcileModFiles must NOT move or disable
  the installed mods.
